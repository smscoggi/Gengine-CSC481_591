Features of the SGG Engine!

Sprites:

SGG Engine allows for easy to upload sprites through links, with easy to manipulate attributes such as rotation, id, even velocity. Two arrays already defined hold your sprites: sprites, and drawnSprites. The sprites array holds all original sprites, while those in drawnSprites act as changeable and updatable copies automatically drawn to your Canvas! In addition, it has a variety of functions for the purpose of simple sprite manipulations such as findSprite().

Collision Detection:

Collision detection follows a component based architecture in which you add a collider object to your sprites and other objects as needed. You decide what shape it supports, bounding boxes or circles, and what happens upon collision.

Particle system:

Our particle system lets you define randomly generating particles with preset attributes that you can control and manipulate, allowing you to add anything from randomly moving asteroids to smoke.

Input:

Current support for the ‘wasd’ keys and arrow keys input as well as mouse events,  ‘Esc’ button events and ‘spacebar’ button events. Input events provides automatic updates to various booleans that you can use, as well as our predefined variable ‘direction’.

Debug system:

SGG Engine has built in functions to check what sprites are in the sprite array or drawnSprites array. This allows a developer to easily check what sprites are being added and what sprites are being drawn.

Tile and Grid System:

A grid is automatically set up so that sprite and other object positions and sizes can be formatted relative to the canvas size or your camera. The grid has presets that can be changed. Grid variables are also used for easy movement updates such as automatic jumping to the other side of the screen or determining routes for AI’s. Tiles can be defined based on the grid positions, and be used for AI movements, collision, and other uses.

AI:

Develop smart AI’s that implement an A* algorithm to efficiently make decisions on how to get to its goal. The A* algorithm is based on using the grid variables.

Networking:

SGG Engine supports peer to peer implementation for developers. Games made by developers can be played online through peer to peer support. One player will host the game and the other player(s) will join the host.

Menu:

There is a default menu screen that the developer can use as a title screen for their game. The developer can customize the menu to their liking.

Physics:

Other than collision, there are various physics capabilities that you can take advantage of, such as velocity and momentum manipulation, distance checking, and rotation.
